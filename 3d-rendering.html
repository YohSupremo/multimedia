Production Breakdown

01

Scene Cleanup

When opening Blender, a default scene is generated containing a Camera, a Point Light, and a default Cube. To start clean, select the Cube in the center of the viewport (or from the Scene Collection Outliner in the top-right corner) and press the Delete key (or press X and click delete) to remove it. This ensures we start with a clean workspace.

Cleanup
02

Create UV Sphere Mesh

Open the Add Menu by pressing Shift + A on your keyboard. In the popup menu, hover your mouse over the Mesh category and click on UV Sphere. This generates a standard spherical mesh at the 3D cursor (coordinates 0, 0, 0) which will act as the baseline geometry for our planet.

Mesh
03

Initialize Material Slot

Look at the properties inspector column on the right side of the screen. Click on the Material Properties icon (represented by a red checkered sphere icon near the bottom of the inspector column). Click the New button at the top of this panel to initialize a new material slot and assign a default Principled BSDF shader to the sphere.

Shading
04

Select Texture Input

Once the new material is created, its shader properties will appear below. Locate the Base Color property. Instead of choosing a solid color, click on the small yellow dot next to the color input field to open the texture type menu, and select Image Texture from the dropdown list.

Texturing
05

Load External Map Image

A new set of options for the Image Texture node will appear. Click the Open button next to the image selector. The Blender File View window will pop up. Navigate through your local files, choose the high-resolution texture map you downloaded for your chosen planet (for example, a Mars diffuse map), and click Open Image to load it.

Texturing
06

View Textured Model

To preview the texture mapped onto the UV sphere in real-time, press and hold the Z key to open the shading pie menu. Move your mouse cursor towards the Rendered option (at the top of the pie menu) and release the key. The viewport will switch to render preview mode, showing the textured sphere.

Preview
07

Rename Mesh Object

In the Outliner panel (located in the top-right corner of the workspace), double-click the object named "Sphere". Rename it to a descriptive title that represents your planet (e.g. "Mars" or "Mercury"). Proper naming is critical because the custom Python automation script relies on these exact object names to target and animate them.

Scene
08

Switch to Scripting tab

Switch your workspace layout to scripting mode. In the top application header ribbon, click on the Scripting tab. This rearranges the Blender interface, opening a Text Editor, a Python Console, and keeping a 3D viewport visible on the side.

Scripting
09

Compile and Play Script

In the Scripting Text Editor, click the New button to create a blank text block. Click inside the text name field and rename it to something descriptive (e.g. planet_anim.py). Write or paste your custom Python animation script utilizing Blender's bpy module, and then click the Run Script button (triangular play icon in the text editor header) to execute the code.

Scripting
10

Confirm & Save Progress

Switch back to the default Layout workspace tab in the top ribbon to view your scene. You will observe the changes applied by the script (for example, a subdivision surface modifier applied to smoothen Mercury). To preview any orbital or rotational animation keyframed by the script, press the Spacebar or click the play button in the timeline at the bottom. Save your project using Ctrl + S.

Finalize
01

Download Addon Package

Visit the official BlenderKit website (blendkit.com) and click the download button to save the addon package to your computer. The download will be a compressed .zip file. Do not unzip this folder — Blender's addon installer requires the zip file directly.

Resources
02

Empty Workspace Scene

Launch Blender on your computer. To avoid clutter, select the default objects (the default cube, camera, and light) and remove them by pressing X and clicking Delete, preparing your workspace for the new library objects.

Scene
03

Open Preferences Inspector

Navigate to the top-left main menu header, click on Edit, and select Preferences from the bottom of the dropdown menu to open the Blender Preferences settings window.

System
04

Install Addon File

In the Preferences window, select the Add-ons category on the left. Click the Install... button (or the dropdown arrow at the top right of the Preferences panel depending on your Blender version). The Blender File View will pop up; navigate to where you saved the downloaded BlenderKit .zip file, select it, and click Install Add-on.

System
05

Enable Online Asset Library

Locate the newly installed 3D View: BlenderKit Online Asset Library addon in the list. Check the checkbox next to its name to enable the online asset search and database connections, then close the Blender Preferences window.

Setup
06

Reveal Interface Sidebar

In the standard 3D Viewport Layout window, press the N key on your keyboard (or click the tiny arrow handle tab located at the upper-right border of the 3D viewport) to toggle open the properties sidebar.

Interface
07

Import Addon Materials

In the newly revealed sidebar, select the BlenderKit tab. Type search keywords for whatever assets or materials you need (e.g. "Space Texture" or "Procedural Planet Material") in the search bar and press Enter. Click on the material or asset thumbnail you want, and wait a moment for the addon to download and automatically assign it to your selected scene object.

Texturing
08

Inspect in Shaded View

Once the asset finishes downloading, it will be placed in your scene. To inspect the full shader details, hold down the Z key and move your cursor to Rendered mode to view the textured materials with lights and shadows. This asset is now integrated into your Blender project and can be controlled or animated by running scripts.

Project Notes

Project goal: Recreate a visually accurate Solar System in 3D using Blender 5.1, automating the scene construction with a custom Python script via the bpy API — covering scripted modelling, texturing, lighting, orbital animation, and real-time EEVEE rendering.

Challenges: Managing extreme scale differences between planets while keeping all objects visible in one scene required creative proportional scaling in the script. Configuring EEVEE for realistic lighting — including screen-space reflections, bloom, ambient occlusion, and getting Saturn's ring transparency to blend correctly in a real-time rasterizer — took the most iteration.

Learning outcomes: Gained hands-on experience with Blender 5.1's Python API (bpy) for procedural scene generation, shader node automation, and real-time render configuration. The project covered the full pipeline from scripted mesh creation through EEVEE rendering (calibrated to 16 viewport samples and 128 render samples), compositing, and final video export.

Time spent: Approximately 10–12 hours across script development, material adjustments, lighting tweaks, and rendering sessions.

The Sun

Star · G-type

The Sun is the star at the center of our Solar System — a nearly perfect sphere of hot plasma that accounts for 99.86% of the total mass of the Solar System. Its core reaches temperatures of 15 million °C, fusing hydrogen into helium and releasing the energy that powers all life on Earth.

Surface Temp~5,500°C
Core Temp~15 million °C

The Moon

Natural Satellite

Earth's only natural satellite and the fifth-largest moon in the Solar System. It stabilizes Earth's axial tilt, drives ocean tides, and is the only extraterrestrial body humans have set foot on — first reached during Apollo 11 in 1969.
